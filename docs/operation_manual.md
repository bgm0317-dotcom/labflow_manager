# Axiom LabFlow 활용 매뉴얼

Axiom LabFlow를 이용한 실험 관리 및 데이터 기록 방법 표준 프로세스입니다.

---

## 1. 일일 루틴

### 1.1 로그인 및 데이터 동기화
1.  @macrogen.com 구글 계정으로 로그인하고, Axiom 앱에 대한 모든 권한을 허용합니다.
2.  **데이터 동기화:** GWS 내의 지정된 파일 기반으로 데이터가 동기화되며, `Refresh` 버튼을 클릭하면 새로고침됩니다.

### 1.2 일일 업무 리스트 (Daily task list)
1.  고정 팀원은 코드 내에 하드코딩되어 있으며, `Add members` 버튼을 클릭하여 새 멤버를 임시로 추가할 수 있습니다.
2.  `+` 버튼을 눌러 공정별 업무를 배정합니다.
3.  Calendar 아이콘을 누르면 해당 팀원의 실험 스케줄과 해당 일자의 Wash 스케줄에 맞게 Google calendar에 일정이 추가됩니다.
4.  휴지통 아이콘을 누르면 해당 팀원의 업무가 전체 삭제됩니다.
5.  본인에게 배정된 업무를 수행 완료 시 체크박스를 클릭하면 완료 시각이 기록됩니다.
6.  해당 업무 내역과 진행 상황은 firebase에 저장되며 `Reset` 버튼을 눌러야 초기화됩니다.

---

## 2. 실험 단계별 데이터 기록 프로세스

### 2.1 Day 1: 배치 등록 및 칩 정보 추가
1.  실험 포맷(96/384)을 선택하고, 전체 #p를 이어서 입력합니다. (입력 예시: P1-2, X3-6)
2.  `저장` 클릭 시, `Weekly schedule` 시트의 오늘 날짜 컬럼에 #p가 입력되고 `Chip info` 마스터 시트에 #p가 parsing되어 신규 행이 자동으로 추가됩니다.

### 2.2 Hyb: 시각 기록
1.  각 슬롯별(🌅, 🌞, 🌙) Hyb 완료 시각을 기록합니다.
2.  입력된 시간은 구글 시트의 해당 시간 필드에 즉시 동기화됩니다.

### 2.3 시약 로그 기록
1.  시약 준비 시, **Reagent**를 선택합니다.
2.  준비된 시약의 Lot 번호를 입력(카메라 스캔 지원)하고, 체크리스트를 모두 완료합니다.
3.  `Reagent log` 시트에 작업 내역이 누적됩니다.

---

## 3. 품질 및 이슈 관리 (Quality & Issue)

### 3.1 칩 이미지 및 일반 이슈 등록
1.  실험 중 문제 발생 시 **Issues** 탭으로 이동합니다.
2.  이슈 내용을 구체적으로 작성하고, `#p` (플레이트 번호)를 정확히 명시합니다.
3.  **사진 업로드:** 카메라 아이콘을 클릭하여 현장의 특이사항을 즉시 촬영/업로드합니다. 구글 드라이브의 지정된 폴더에 이슈 날짜를 이름으로 갖는 폴더가 생성되고, 그 안에 사진이 저장되어 확인 가능합니다.

### 3.2 칩 QC 완료 처리
1.  **Dashboard**에서 `Chip QC 미완료` 개수를 확인할 수 있습니다.
2.  `Chip info` 마스터 시트에 CHQ 완료 일자가 등록되면 **Dashboard**에도 반영됩니다.
3.  납기가 임박한 chip이 있는 경우 `Chip QC 미완료` 박스에 애니메이션이 적용됩니다.
